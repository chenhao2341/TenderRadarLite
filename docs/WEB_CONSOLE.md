# TenderRadarLite Web 控制台

## 1. 当前定位

TenderRadarLite Web 控制台当前仍是 Alpha / 本地控制台骨架。

它的作用是提供一个轻量浏览器入口，用来查看：

- Dashboard 状态摘要
- 推荐运行命令
- 报告入口
- 日志摘要
- 配置状态
- Source Catalog 只读视图

它不是成熟控制台，不是 SaaS 后台，也不是完整运行入口。

## 2. 当前能做什么

- 查看本地项目状态
- 查看 `reports/latest.html` 是否存在
- 打开报告入口
- 查看 `logs/` 摘要
- 查看配置是否存在
- 查看来源目录状态

## 3. 当前不能做什么

- 不能替代命令行完成完整运行闭环
- 当前缺真实运行按钮
- 当前缺完整运行历史
- 当前缺来源启停可视化写回
- 当前缺稳定的错误处理工作台

因此，当前用户主路径仍应是：

- 命令行
- Windows `.bat`
- 本地 HTML 报告

## 4. 如何启动

```powershell
python scripts/start_web_console.py
```

或双击：

```text
启动Web控制台.bat
```

默认地址：

```text
http://127.0.0.1:8765
```

## 5. 安全默认值

当前页面默认不会触发 Feishu。

- 不会自动发送 webhook
- 不会自动写入飞书多维表
- 不会自动触发真实 Feishu 流程

当前页面默认不会调用 AI。

- 推荐命令默认只包含 `--local-html`
- 默认不附带 `--ai-analysis`
- 页面不主动触发模型请求

## 6. 报告与日志

当前控制台可以帮助用户找到：

- `reports/latest.html`
- `logs/`

但它不是“点击一下就完成一次真实运行”的成熟入口。

如果 `reports/latest.html` 不存在，仍应回到命令行执行：

```powershell
python run_mvp.py --local-html --profile design_consulting
```

## 7. Source Catalog 页面边界

`/sources` 页面当前是只读来源目录视图。

它可以解释：

- 哪些来源是 `supported`
- 哪些来源是 `alpha`
- 为什么部分来源默认关闭

它不能：

- 新增来源
- 编辑来源
- 删除来源
- 直接运行来源

## 8. 为什么当前不把它包装成成熟控制台

因为当前版本仍缺：

- 真实运行按钮
- 完整运行闭环
- 运行历史
- 来源启停可视化主流程
- 错误日志统一展示与回溯

把当前骨架包装成成熟控制台，会误导外部对项目阶段的判断。

## 9. 下一阶段重点

`v0.2-alpha` 推荐重点是：

1. Web 控制台真实运行入口
2. 运行历史
3. 来源启停可视化
4. 错误日志展示
5. 报告入口整合

也就是说，Web 控制台会继续做，但它是下一阶段重点，不是当前 `v0.1-alpha` 已完成能力。

## 10. 当前结论

如果要对外描述当前状态，最准确的说法是：

- Web 控制台当前是本地 Alpha 控制台骨架
- 当前用户仍以命令行 / bat / 本地 HTML 报告为主路径
- 真实运行入口是 `v0.2-alpha` 的优先任务
